## 项目简介

本仓库是南京大学软件学院《互联网计算》课程的改进计划的内部仓库，视情况发布后会公开。

本仓库的文档组织使用了VuePress的形式，以方便版本控制以及发布Web版本。实际线下导出时仍然使用PDF这样的形式。PDF内尽量使用PDF自带的标签实现目录跳转，最后如需导出纸质版，则使用手动目录的方式导出。后期也可能采取GitBook这样的形式导出为真正的书的形式。不过反正Markdown文档里面其实也就具体的Markdown内容，实际上放哪里并没有什么区别。

## 贡献指南

如果要看开发版本的网页，请先安装相关依赖。YDJSIR使用的node版本是v14.18.3，Windows 64位。本项目使用yarn来管理依赖，请先用npm安装yarn。

```
yarn install
```

接下来，如果你要预览网页版，请使用如下命令。

```
yarn dev
```

在这样的状况下，对Markdown文本本身的更改会同步到网页上。网页会自动刷新。但要注意，侧边栏的索引是需要先把当前执行结束，再次执行此命令才能看到更新的。

最后，如果你要输出最终结果的内容，请执行如下内容。

```
yarn build
```

输出后，请到 `.vuepress/dist` 下找到可以对外部署的网页版本。

```
yarn vuepress export src
```

最后，你会在根目录找到输出的 `site.pdf` 。这个PDF的输出顺序很有问题，也没页码，最后发布的时候可能还要手动更新一下。
很明显，这些输出的内容，都是被gitignore的。


关于图片，请把它放在文件名`.assets`文件下。请使用以`./`开头的相对路径。可以用Typora来管理。
